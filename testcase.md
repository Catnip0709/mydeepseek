**十九、边界与异常流程**
- 页面在没有任何会话内容时能正常工作
- localStorage 中旧数据结构仍可兼容读取
- 老会话数据中存在旧 `memoryLimit` 字段时不影响当前逻辑
- 网络异常时，页面不会整体卡死
- API 返回错误时，会显示错误信息
- 失效 Key、网络错误、后台切回等异常路径下，不应破坏已有会话内容
- 刷新页面后，会话、设置、指令、主题等本地数据可恢复
- 多次快速切换设置项，不应导致明显状态错乱

**二十、存储用量显示与保护**

- 存储用量在发送消息、保存指令、保存 API Key 后自动更新
- 存储用量 < 95% 时显示灰色文字，无感叹号图标

- 点击感叹号图标弹出提醒："当前聊天内容接近本地存储上限，请及时导出并清理过期会话。"
- 存储用量 ≥ 99% 时，点击发送按钮弹出阻止提示，不允许发送新消息
- 存储用量 ≥ 99% 时，按回车同样被阻止
- 存储用量在日间模式下正常显示（灰字/红字适配）

**二十一、数据损坏自动修复**
- `dsTabs` 数据损坏时，页面自动修复并刷新，不会白屏
- `dsTabs` 数据损坏时，修复逻辑会尝试补齐缺失字段（title、memoryLimit）
- `dsTabs` 数据彻底损坏时，清除并重置为空白状态
- `dsPrompts` 数据损坏时，自动重置为空数组，指令管理功能正常
- `MARKET_PROMPTS` 加载失败时，指令市场按钮不崩溃，显示空列表

**二十二、CDN 资源加载失败保护**
- `marked.js` 加载失败时，聊天消息降级为纯文本显示，不崩溃
- `DOMPurify.js` 加载失败时，聊天消息降级为纯文本显示，不崩溃
- CDN 加载失败后，发送消息、流式输出等核心功能仍可正常使用

**二十三、请求超时控制**
- 聊天请求 120 秒无响应后自动中断，显示"请求超时，请检查网络后重试"
- 指令生成请求 60 秒无响应后自动中断
- 超时中断后，发送按钮恢复正常状态（不再显示"停止"）
- 正常请求（非超时）不受影响

**二十四、Tab DOM 缓存与性能优化**
- 切换会话后，聊天区正确显示目标会话内容
- 切换会话后，消息的复制、编辑、删除、重新生成按钮均可正常点击
- 切换会话后，版本切换按钮（❮ ❯）可正常使用
- 切换会话后，token limit 提醒中的"复制指令"按钮可正常使用
- 在会话 A 中删除消息后，切到会话 B 再切回 A，显示的是删除后的最新内容
- 在会话 A 中切换版本后，切到会话 B 再切回 A，显示的是切换后的版本
- 搜索功能执行后，Tab DOM 缓存被正确清除，搜索结果正常显示
- 关闭搜索后，聊天内容正常恢复（无残留高亮）

**二十五、Markdown 渲染缓存**
- 相同内容的消息多次渲染时，显示结果一致
- 搜索高亮模式下，缓存不影响高亮显示的正确性
- 长对话场景下，切换会话的响应速度明显快于全量渲染

**二十六、用户行为追踪**
- 页面加载时会自动生成 `ds_user_id` 并写入 localStorage
- 刷新页面后 `ds_user_id` 保持不变
- 页面加载时会发起一次 webhook 请求（访问事件）
- 发送消息时会发起一次 webhook 请求（发送事件）
- webhook 请求失败不影响正常功能

**二十七、真实 API 冒烟测试**
- 使用真实 Key 发起最小 `stream: false` 请求，验证返回内容和 `usage`
- 使用真实 Key 发起最小 `stream: true` 请求，验证 SSE 分块可收到
- `deepseek-chat` 普通请求可用
- `deepseek-reasoner` 流式请求可用
- 使用最短输入和较小 `max_tokens`，避免额外消耗过多额度
